## â˜ï¸ Cloud Sync (Optional) / äº‘ç«¯åŒæ­¥ (å¯é€‰)

Sync your library across devices using your own backend (**Supabase**). Privacy-first.
ä½¿ç”¨æ‚¨è‡ªå·±çš„åç«¯ (**Supabase**) åœ¨å¤šè®¾å¤‡é—´åŒæ­¥æ–‡çŒ®åº“ã€‚éšç§ä¼˜å…ˆã€‚

1.  **Create Project**: Go to [Supabase](https://supabase.com) and create a free project.
2.  **Get Credentials**: In Supabase Dashboard -> Project Settings -> API, copy the **URL** and **anon public key**.
3.  **Setup Database**: Go to **SQL Editor** in Supabase and run the following script:

```sql
-- Create the papers table
create table papers (
  id uuid primary key,
  user_id uuid references auth.users not null,
  title text,
  original_title text,
  source_url text,
  storage_path text,
  analysis jsonb,
  tags text[],
  status text,
  is_favorite boolean default false,
  user_notes text,
  created_at timestamptz default now(),
  updated_at timestamptz default now()
);

-- Enable Security (RLS)
alter table papers enable row level security;

-- Create Storage Bucket for PDFs
insert into storage.buckets (id, name, public) values ('papers', 'papers', false);

-- Policies (Allow users to manage their own data)
create policy "Users can manage their own papers"
on papers for all
using (auth.uid() = user_id);

create policy "Users can upload their own pdfs"
on storage.objects for insert
with check ( bucket_id = 'papers' and auth.uid()::text = (storage.foldername(name))[1] );

create policy "Users can view their own pdfs"
on storage.objects for select
using ( bucket_id = 'papers' and auth.uid()::text = (storage.foldername(name))[1] );
```

4.  **Connect**: Enter credentials in ScholarSense Settings -> Cloud Sync, and Sign Up/Login.
